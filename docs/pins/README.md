# Pinned Notes

This folder is the default home for durable project context that should stay easy to find across sessions.

Use it for notes that do not belong directly in a requirement body or `CHANGELOG.md`, but still matter enough to keep nearby and reviewable.

Good pin candidates:

- decisions that affect future implementation direction
- quick-reference project context that would otherwise disappear into chat history
- reminders about conventions, constraints, or repository-specific expectations

Suggested workflow:

1. Start from [pin-template.md](pin-template.md).
2. Create one note per topic.
3. Add a short bullet to the index below.
4. Use `/rqmd-docs` later if this area grows enough to need better structure or splitting.
